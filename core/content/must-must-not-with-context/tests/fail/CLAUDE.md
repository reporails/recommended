# Project Instructions

## Code Quality

You MUST validate all user input.
You MUST NOT skip type checking.
You MUST write tests for public functions.

## Security

You MUST encrypt sensitive data at rest.
You MUST NOT log personally identifiable information.
You MUST rotate API keys quarterly.

## Git Workflow

You MUST use feature branches for all changes.
You MUST NOT force push to shared branches.
You MUST include a ticket reference in commit messages.

## Deployment

You MUST run the full test suite before deploying.
You MUST NOT deploy on Fridays.
You MUST tag every production release.

## Documentation

You MUST update the changelog for user-facing changes.
You MUST NOT leave TODO comments in merged code.

## Dependencies

You MUST pin exact dependency versions.
You MUST NOT use deprecated APIs.
